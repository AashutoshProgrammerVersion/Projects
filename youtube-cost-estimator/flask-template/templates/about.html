Tech Stack

This project is built using:

  • Flask: A lightweight Python web framework chosen for its simplicity, single-purpose application and rendering results.
  • YouTube Data API: For fetching reliable channel statistics and analytics directly from YouTube.
  • Google Gemini AI: To compare my program analysis to AI to forge a better conclusion.
  • HTML/CSS: Core functionality for the text adn its structure and I can use AI for it to make the frontend look nice easily.

Estimation Methodology

Our pricing algorithm considers three key factors:

  • Audience Size Impact: AI did this: average views per video as the base metric, with a logarithmic scale to prevent overvaluation of viral content; it was implemented in the recent views multiplier
  • View Duration: Initially AI used subscribers and views, but as a content creator, I thought it is better to measure view duration, making the price go up to 2x for channels with high retention rates.
  • Subscriber Influence: AI told me to use a logarithmic scale to account for channel authority while preventing excessive inflation for mega-channels.
  • Calculation for price Use of multipliers accounts for the various factors

Use of CPM rates as a baseline as I learned about it watching videos about YouTube and know that is how the pay of a YT is measured

Technical Challenges

The main challenge was handling various YouTube channel URL formats and balancing use of AI with still coding my project myself:

  • YouTube channels can have multiple URL formats: /channel/UC..., /c/channelname, /@username
  • Solution: With the help of AI, I implemented regex pattern matching to extract channel identifiers from any valid YouTube channel URL format.
  • Additional challenge was obtaining accurate view duration data, which isn't directly available through the API. I overcame this by considering how average view duration as a percentage tends to decrease with longer videos.
